Read a (B)ounded (I)nteger (L)inear (P)rogramming problem

The format of a BILP is a sequence of lines of the form: 

l <= a_1 a_2 ... a_n <= u

where l, u, a_1, a_2, ..., and a_n are integers that are separated by whitespace.

TODO: support rational numbers

in - (input) input stream to parse L - (output) lattice matrix where rows are lattice vectors C - (output) constraint matrix

Read a lattice in "Boost" format from file into a matrix<int64>

The result is stored in the output parameter 'm' passed by reference. Returns 0 on success, or 1 on read or parse error.

The matrix format is e.g., 

[2,2]
((1,0), (0,1))

See documentation in boost/numeric/ublas/io.hpp for more details.
